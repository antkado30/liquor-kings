# Order Day — Thursday July 2, 2026 (Colony, 8pm ET cutoff)

Two goals in one run: (a) place Colony's real weekly order IN-APP (real-order
attempt toward the 3-order mandate), (b) capture MILO's submit/checkout
endpoint via the run's network recording — the last missing piece for the
fast engine.

Every command below is exact — no placeholders. Do them in order.
Full arming context: `order-submission-go-live.md` (same folder).

---

## 0 — Morning health check (before anything)

```
node services/api/scripts/order-day-preflight.mjs
```

One command, GO / NO-GO verdict: both apps healthy, both env gates
disarmed (correct for the morning), capture switch on, worker at exactly
1 machine, Colony store flag true. Any ✗ comes with the exact reason —
paste the output to Claude if it's not GO.

(Manual fallback: `fly status -a liquor-kings` + same for the worker.)

## 1 — Capture switch (FIRST, while no check is running)

```
fly secrets set LK_RPA_PERSIST_SESSION=no -a liquor-kings-worker
```

This is what makes the recording actually save to disk (non-persisted
teardown closes the browsing context → Playwright flushes `network.har`).
Skipping this = no capture, whole second goal lost.

The worker restarts on this command (~1 min). If a check was mid-flight it
dies and blocks the store with "a check is already running" — it self-heals
in ~3 minutes (reaper). Side effect all day: practice checks do a cold
login each run, so they'll take ~30-60s instead of ~5-15s. Expected, not a
bug.

## 2 — Arm the four locks (in this order)

1. In `apps/scanner/src/config/submission.ts` set
   `export const REAL_SUBMISSION_WIRED = true;`
   then commit and `npm run deploy`
   (this deploy also ships the already-committed scanner instant-refresh
   `34d8f00` — the planned batch).
2. `fly secrets set LK_ALLOW_ORDER_SUBMISSION=yes -a liquor-kings`
3. `fly secrets set LK_ALLOW_ORDER_SUBMISSION=yes -a liquor-kings-worker`
4. Colony store flag — run in the Supabase **prod** SQL editor
   (project `eamoozfhqolshdztbrez`):

```sql
update stores set allow_order_submission = true
where id = 'e594fc3a-17b7-45d0-9dde-943ebbfa5391';
```

## 3 — Verify ARMED before placing

```
node services/api/scripts/order-day-preflight.mjs --expect armed
```

Must say GO. Then the phone check (the one thing the script can't see —
the deployed client flag): kill + reopen the scanner → cart → tap Place
Order. The confirm modal MUST say **"This goes to MILO immediately and
can't be unsent"** with button **"Confirm & send to MILO"**. If it still
shows practice copy ("Preview only…"), STOP and tell Claude.

## 4 — Place the order (afternoon — don't race the 8pm cutoff)

Recommended rhythm (Tony's two-step, no code needed): build the cart
while still DISARMED → tap "Check Order" → read MILO's answer on the
sheet (out-of-stocks, totals) → fix anything → THEN arm (step 2's env
secrets) → verify (step 3) → place the same cart.

Optional: watch the run live in a second terminal —

```
fly logs -a liquor-kings-worker
```

The log line `[worker] Stage 5 arming: … finalMode=submit` is the
moment-of-truth record of the arming decision.

Build the real cart → Place Order → confirm → pill shows "Placing your
order" → wait for terminal (~3-6 min; cold login + full browser path —
that's what records the capture).

- SUCCESS = pill "Order placed" + a confirmation number. Cross-check on
  MILO: `/milo/account/orders`.
- **If it says FAILED after the confirm: DO NOT tap again.** Check MILO's
  orders page first — the order may have been placed even if the app
  couldn't read the confirmation. The duplicate-submit tripwire blocks
  re-submits for 30 min for exactly this case. Tell Claude what MILO shows.

## 5 — Pull the capture IMMEDIATELY

Right after the order reaches terminal — **before running any more checks
and before touching any secrets** (a later check shadows the newest
recording; a worker restart can wipe it):

```
node services/api/scripts/pull-latest-har.mjs
```

Saves to `./rpa-captures/`. Claude then verifies the checkout call is in
it and extracts MILO's submit endpoint. Only disarm after Claude confirms
the capture is good.

## 6 — Disarm (after capture confirmed + order verified on MILO)

```
fly secrets set LK_ALLOW_ORDER_SUBMISSION=no -a liquor-kings-worker
fly secrets set LK_ALLOW_ORDER_SUBMISSION=no -a liquor-kings
fly secrets set LK_RPA_PERSIST_SESSION=yes -a liquor-kings-worker
```

`REAL_SUBMISSION_WIRED` can stay `true` — with the env gates off, the app
shows practice copy and every run dry-runs (verified: armed copy requires
env + store flag + the code flag together). Last command restores warm
fast validates.
