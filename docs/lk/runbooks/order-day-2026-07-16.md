## 1 — Capture switch (FIRST, while no check is running)

```
fly secrets set LK_RPA_PERSIST_SESSION=no -a liquor-kings-worker
```

This is what makes the recording save to disk (non-persisted teardown
closes the browser context → Playwright flushes `network.har`). Skipping
this = no capture, second goal lost.

Worker restarts on this (~1 min); a mid-flight check would die and
self-heal in ~3 min (reaper). Side effect all day: every check does a
cold login, so checks run ~35-60s instead of ~20s. Expected, not a bug.

## 4 — Place the order (afternoon — don't race the 8pm cutoff)

The two-step rhythm, now built into the UI:

1. Build the real cart at the shelf (scan / photo-to-AI / search — note
   anything the AI misses for the resolver list).
2. Tap **Check with MLCC** → drawer closes, pill tracks it → banner
   lands (~35-60s cold). Read the sheet: out-of-stocks, totals, per-ADA.
3. Fix anything (evict OOS, adjust) → check again if you edited —
   Place re-locks on any change, by design.
4. When the check is green, reopen the cart: **Place Order is lit**,
   with "Checked N min ago — cart matches MILO." You have 10 minutes;
   past that it re-locks and one more Check re-arms it.
5. Tap Place → the line-by-line confirm modal ("This goes to MILO
   immediately and can't be unsent") → **Confirm & send**.

Optional second terminal to watch it live:

```
fly logs -a liquor-kings-worker
```

`[worker] Stage 5 arming: … finalMode=submit` is the moment-of-truth log
line. The submit run takes ~3-6 min (cold login + full browser path —
that's exactly what records the capture).

- SUCCESS = pill "Order placed" + confirmation numbers on the sheet.
  Cross-check on MILO: `/milo/account/orders`.
- **If it says FAILED after the confirm: DO NOT tap again.** Check
  MILO's orders page first — the order may exist even if the app
  couldn't read the confirmation. The duplicate-submit tripwire blocks
  re-submits for 30 min for exactly this case. Tell Claude what MILO
  shows.
